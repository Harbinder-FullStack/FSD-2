Lecture No. 3: State Management (Redux / Context API)

1. Introduction to State

In modern Single Page Applications (SPAs), state represents the data that determines how a user interface behaves and what it displays at any given time.

Examples of state:

  • User login status
  • Theme (light/dark)
  • Shopping cart items
  • Form input values

2. Types of State

2.1 Local State

  • Managed inside a single component
  • Implemented using useState
  • Best for component-specific data

2.2 Global State

  • Shared across multiple components
  • Centralized data storage
  • Used for authentication, cart, preferences

3. Problems Without State Management

  • Prop Drilling – passing data through many components
  • Unpredictable UI – hard to track changes
  • Poor scalability – difficult to maintain large apps

4. Redux Overview

Redux is a predictable state container for JavaScript applications. It enforces a strict unidirectional data flow.

Redux Core Principles

  1. Single source of truth
  2. State is read-only
  3. Changes via pure functions (reducers)

5. Redux Architecture Flow

Flow Diagram:

User Action → Dispatch Action → Reducer → Store → UI Update

  • Action: Describes what happened
  • Reducer: Updates the state
  • Store: Holds the global state

6. Redux Toolkit

Redux Toolkit is the official, recommended way to use Redux.

  • Less boilerplate
  • Built-in best practices
  • Immutable updates handled automatically

7. Context API

Context API is a built-in React feature used to share data across components without prop drilling.

Context Flow

Provider → Consumer Components

8. Redux vs Context API

Feature Redux Context API
Application Size Large / Complex Small / Medium
Performance Highly optimized Limited for frequent updates
Debugging Excellent (DevTools) Basic
Learning Curve Moderate Easy

9. Best Practices

  • Keep state minimal
  • Avoid unnecessary global state
  • Use Redux Toolkit instead of classic Redux
  • Separate UI state and business logic

10. Summary

State management is essential for scalable front-end
