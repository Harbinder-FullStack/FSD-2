Lecture No. 1: Modern Front-End Architecture & SPA Concepts

1. Introduction

Modern web applications are expected to be fast, interactive, and user-friendly. Traditional web architectures struggle to meet these expectations. As a result, modern front-end frameworks and Single Page Application (SPA) architecture have become industry standards.

2. Traditional Multi-Page Applications (MPA)

In a Multi-Page Application, each user request results in a full page reload. The server generates a new HTML page for every navigation or form submission.

Characteristics of MPAs

  • Full page reload on every request
  • Server-side rendering of HTML
  • Tightly coupled frontend and backend
  • Slower user experience for interactive systems

Request Flow:

User Action → Server Request → HTML Response → Full Page Reload

3. Single Page Applications (SPA)

3.1 What is a SPA?

A Single Page Application loads a single HTML page once and dynamically updates the content using JavaScript without reloading the entire page. Only data is exchanged with the server, usually in JSON format.

3.2 SPA Request Flow

Initial Load:
Browser → index.html + JavaScript bundle

After Initial Load:
User Action → JavaScript Logic → Optional API Call → UI Update

3.3 Key Characteristics of SPAs

  • Single HTML entry point
  • Client-side routing
  • API-driven backend communication
  • Faster navigation and better UX

4. Modern Front-End Architecture

4.1 Component-Based Architecture

Modern front-end frameworks break the UI into reusable components. Each component encapsulates its structure, logic, and behavior.

Benefits of Component-Based Design

  • Reusability
  • Modularity
  • Easier maintenance
  • Improved scalability

4.2 Separation of Concerns

Modern front-end applications separate responsibilities across layers to improve maintainability and clarity.

Layer Responsibility
UI Components Rendering and user interaction
Routing Navigation and page mapping
State Management Application data handling
Services API communication

5. Modern Front-End Frameworks

As applications grow in complexity, managing UI updates using plain JavaScript becomes difficult. Modern frameworks provide structure, efficiency, and scalability.

Framework Type
React Component-based library
Angular Full-featured framework
Vue Progressive framework

For this course, React is used because it aligns with the MERN stack and is widely adopted in the industry.

6. Client-Side Routing

In SPAs, routing is handled on the client side. Changing the URL does not reload the page. Instead, the framework decides which component to render.

Example Routes:

  • / → Home Component
  • /about → About Component
  • /contact → Contact Component

7. Entry Point of a SPA

A SPA has a single HTML entry file, usually index.html, containing a root DOM element. The JavaScript framework mounts the application into this root element.

8. Advantages of SPA Architecture

  • Fast navigation without page reload
  • Better user experience
  • Decoupled frontend and backend
  • Reusable UI components

9. Limitations of SPAs

  • Higher initial load time
  • SEO challenges
  • Heavy reliance on JavaScript

These limitations are addressed later using techniques such as code splitting, server-side rendering, and Progressive Web Apps.

10. Learning Outcomes

After this lecture, students should be able to:

  • Explain SPA architecture
  • Differentiate SPA and MPA
  • Understand modern front-end architecture
  • Identify the entry point of a SPA
